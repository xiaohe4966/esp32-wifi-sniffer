## 一、连接设备

### 1.1 首次连接

设备上电后会自动创建一个 WiFi 热点：

| 配置项 | 值 |
|--------|-----|
| SSID | `ESP32-Sniffer` |
| 密码 | `12345678` |

使用手机或电脑连接该热点，然后打开浏览器访问：

```
http://192.168.4.1
```

### 1.2 界面概览

Web 界面包含 5 个主要标签页：

```
┌─────────────────────────────────────────────┐
│  🔍 ESP32 WiFi Sniffer                      │
│  [Connected] [IDLE]  [EN | 中文]             │
├─────────────────────────────────────────────┤
│ [Dashboard] [Networks] [Capture] [Attack] [Logs] │
└─────────────────────────────────────────────┘
```

- **Dashboard** - 设备状态和快速操作
- **Networks** - 扫描到的 WiFi 网络列表
- **Capture** - 数据包捕获和握手包抓取
- **Attack** - Deauth 攻击和字典攻击
- **Logs** - 运行日志

## 二、基本操作流程

### 2.1 扫描 WiFi 网络

**步骤：**

1. 切换到 **Dashboard** 标签页
2. 点击 **Start Scan** 按钮
3. 等待 10 秒左右，设备会自动在所有信道间跳转扫描
4. 切换到 **Networks** 标签页查看结果

**Networks 页面说明：**

| 列名 | 含义 |
|------|------|
| SSID | WiFi 名称（隐藏网络显示为空） |
| BSSID | 路由器的 MAC 地址 |
| Channel | 信道号（1-14） |
| RSSI | 信号强度（负数，越接近 0 越强） |
| Security | 加密类型（OPEN/WPA2/WPA3 等） |
| Packets | 捕获的数据包数量 |

**操作按钮：**

- **Select** - 选中该网络，自动填充到 Capture/Attack 表单
- **Capture** - 立即开始抓取该网络的握手包

## 三、握手包捕获（重点）

要破解 WPA/WPA2 密码，需要先捕获**握手包（Handshake）**。

### 3.1 什么是握手包？

当设备连接 WiFi 时，会进行 4 次握手（M1、M2、M3、M4）。握手包包含了加密密码所需的信息，可以用于离线破解。

### 3.2 捕获步骤

**方法一：自动捕获**

1. 在 **Networks** 页面找到目标网络
2. 点击该网络的 **Capture** 按钮
3. 等待有设备连接/重连该 WiFi
4. 观察 **Capture** 页面的 M1-M4 指示器

**方法二：手动捕获**

1. 切换到 **Capture** 标签页
2. 在 **Target BSSID** 输入目标路由器的 MAC 地址
3. 点击 **Start** 开始抓包
4. （可选）在 **Attack** 页面发送 Deauth 强制客户端重连

**Capture 页面说明：**

```
┌─────────────────────────────────────────────┐
│ 📦 Capture Control                          │
│ Channel: [All (Hop) ▾]                      │
│ Target BSSID: [AA:BB:CC:DD:EE:FF         ] │
│ [▶️ Start]  [⏹️ Stop]                       │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│ 🤝 Handshake Capture                         │
│ ┌───┐ ┌───┐ ┌───┐ ┌───┐                     │
│ │ M1│ │ M2│ │ M3│ │ M4│  ← 收到变绿色       │
│ └───┘ └───┘ └───┘ └───┘                     │
│ [████████░░░░░░░░] 50%                      │
│ No handshake captured                        │
│ [⬇️ Download]                               │
└─────────────────────────────────────────────┘
```

**M1-M4 指示器：**

| 状态 | 含义 |
|------|------|
| 灰色 | 尚未捕获 |
| 绿色 | 已成功捕获 |

当看到 M1、M2、M3、M4 全部变绿，说明握手包捕获成功！

### 3.3 下载握手包

握手包捕获成功后，点击 **Download** 按钮下载文件。文件格式为 `.hccapx`，可用于 hashcat 或 aircrack-ng 破解。

## 四、字典攻击

### 4.1 前提条件

- ✅ 已成功捕获握手包
- ✅ SD 卡已插入并格式化（FAT32）
- ✅ SD 卡中存有密码字典文件 `wordlist.txt`

### 4.2 准备字典文件

将密码字典文件放入 SD 卡根目录：

```
/sdcard/wordlist.txt
```

每行一个密码，例如：

```
password
12345678
admin123
wifi1234
```

### 4.3 开始攻击

1. 切换到 **Attack** 标签页
2. 在 **Dictionary Attack** 区域：
   - **Handshake File** - 选择握手包文件
   - **Dictionary** - 选择 `wordlist.txt`
3. 点击 **Start Attack**
4. 观察进度条和破解统计

## 五、Deauth 攻击（洪水攻击）

### ⚠️ 警告

Deauth 攻击会强制断开目标网络的所有设备，属于**干扰攻击**：
- 在许多国家/地区，未经授权使用是**违法行为**
- 仅用于授权的安全测试
- 可能影响正常使用网络

### 5.1 用途

Deauth 攻击可以强制目标设备断线，触发其重新连接，从而捕获握手包。

### 5.2 使用步骤

1. 在 **Attack** 页面 **Deauth Attack** 区域
2. 输入目标 BSSID（路由器 MAC）
3. （可选）输入 Target Station 可针对特定设备
4. 点击 **Start Deauth**
5. 观察目标网络的设备是否断线

## 六、高级设置

### 6.1 信道选择

| 选项 | 说明 |
|------|------|
| All (Hop) | 自动在所有信道间跳转 |
| 1-14 | 固定在特定信道 |

固定信道可以减少信道切换开销，提高特定目标的捕获效率。

### 6.2 Reason Code 说明

| Code | 含义 |
|------|------|
| 7 | Class 3 frame from nonassociated STA |
| 6 | Class 2 frame from nonauthenticated STA |
| 1 | Unspecified |
| 2 | Previous auth no longer valid |
| 3 | Disassociated because AP is unable to handle all stations |

通常使用代码 7 效果较好。

## 七、常见问题

### Q1: 扫描不到 WiFi？

- 确保设备天线连接正常
- 靠近路由器再试
- 检查串口日志是否有错误

### Q2: 握手包捕获失败？

- 目标网络需要有设备在线并尝试连接
- 尝试先发送 Deauth 攻击强制重连
- 确保目标不是 5GHz（ESP32-S3 需支持）

### Q3: WebSocket 连接失败？

- 确保连接的是 `192.168.4.x` 网络
- 尝试刷新页面

### Q4: SD 卡无法识别？

- 检查 SD 卡格式（需为 FAT32）
- 检查接线是否正确
